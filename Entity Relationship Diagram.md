# Rising Waters
## Overview

### Entity Relationship Diagram (ER Diagram)

## Story

### Description

The Entity Relationship (ER) Diagram represents the core entities involved in the **Flood Prediction System** and illustrates how they interact with each other. It provides a structured database design for managing users, weather information, machine learning models, and flood prediction results efficiently.

---

## Main Entities

### 1. User

Represents the system users (Admin, Meteorologist, Disaster Management Officer, or Local Authority) who enter weather data and access prediction results.

**Attributes:**
- UserID (Primary Key)
- UserName
- Email
- Password
- Role

---

### 2. Weather Data

Stores weather and rainfall-related information used as input for flood prediction. Each weather record belongs to a specific user.

**Attributes:**
- DataID (Primary Key)
- UserID (Foreign Key)
- AnnualRainfall
- CloudVisibility
- Temperature
- Humidity
- SeasonalRainfall

---

### 3. Machine Learning Model

Stores details of the trained machine learning model used for predicting floods.

**Attributes:**
- ModelID (Primary Key)
- ModelName
- AlgorithmType
- Accuracy
- ModelFile

---

### 4. Prediction Result

Stores the prediction generated by the machine learning model along with flood probability and prediction date.

**Attributes:**
- PredictionID (Primary Key)
- DataID (Foreign Key)
- ModelID (Foreign Key)
- FloodResult
- FloodProbability
- PredictionDate

---

## Relationships

### User → Weather Data
- **Relationship:** One-to-Many (1:N)
- One user can submit multiple weather records.

### Weather Data → Prediction Result
- **Relationship:** One-to-One (1:1)
- Each weather data record generates one prediction result.

### Machine Learning Model → Prediction Result
- **Relationship:** One-to-Many (1:N)
- One machine learning model can generate multiple prediction results.

## Conclusion

This ER Diagram provides a clear and organized database structure for the **Flood Prediction System**. It ensures efficient storage and management of weather data, trained machine learning models, prediction results, and user information while maintaining proper relationships between all entities.
